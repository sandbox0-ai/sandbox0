# Webhooks

Receive real-time event notifications from sandboxes via webhooks. Webhooks allow you to react to process events, sandbox lifecycle changes, and file modifications.

## Webhook Configuration

Configure webhooks when claiming a sandbox:

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Webhook endpoint URL (required) |
| `secret` | string | Shared secret for payload signing (optional) |
| `watch_dir` | string | Directory to monitor for file events (optional) |

<Callout variant="info">
Webhooks are configured per-sandbox at creation time. Each sandbox can have its own webhook configuration.
</Callout>

---

## Event Types

### Process Events

| Event | Description |
|-------|-------------|
| `process.started` | A new process/context has started |
| `process.exited` | A process has exited (includes exit code) |

### Sandbox Events

| Event | Description |
|-------|-------------|
| `sandbox.paused` | Sandbox has been paused |
| `sandbox.resumed` | Sandbox has been resumed |

### File Events

| Event | Description |
|-------|-------------|
| `file.created` | A file was created |
| `file.written` | A file was modified |
| `file.renamed` | A file was renamed |
| `file.chmodded` | File permissions were changed |
| `file.removed` | A file was deleted |

<Callout variant="info">
File events require setting <code>watch_dir</code> in the webhook configuration.
</Callout>

## Webhook Payload Format

### Payload Structure

```json
{
  "event": "process.exited",
  "sandbox_id": "sb_abc123",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "context_id": "ctx_xyz789",
    "exit_code": 0
  }
}
```

### Process Events

```json
// process.started
{
  "event": "process.started",
  "sandbox_id": "sb_abc123",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "context_id": "ctx_xyz789",
    "type": "repl",
    "language": "python"
  }
}

// process.exited
{
  "event": "process.exited",
  "sandbox_id": "sb_abc123",
  "timestamp": "2024-01-15T10:30:05Z",
  "data": {
    "context_id": "ctx_xyz789",
    "exit_code": 0
  }
}
```

### Sandbox Events

```json
// sandbox.paused
{
  "event": "sandbox.paused",
  "sandbox_id": "sb_abc123",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {}
}

// sandbox.resumed
{
  "event": "sandbox.resumed",
  "sandbox_id": "sb_abc123",
  "timestamp": "2024-01-15T10:30:05Z",
  "data": {}
}
```

### File Events

```json
// file.created
{

  }

  "event": "file.written",
```

## Signature Verification

When a secret is configured, webhook payloads are signed using HMAC-SHA256.

### Header

```
X-Sandbox0-Signature: sha256=<hex-encoded-signature>
```

### Verification Example

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `package main

import (
	"crypto/hmac"
	"crypto/sha256"
	"encoding/hex"
	"fmt"
	"log"
	"strings"
)

func verifySignature(payload []byte, signature string, secret string) bool {
	if !strings.HasPrefix(signature, "sha256=") {
		return false
	}
	expectedMAC, err := hex.DecodeString(signature[7:])
	if err != nil {
		return false
	}

	mac := hmac.New(sha256.New, []byte(secret))
	mac.Write(payload)
	actualMAC := mac.Sum(nil)
	return hmac.Equal(expectedMAC, actualMAC)
}

func main() {
	payload := []byte("{\"event\":\"process.exited\",\"sandbox_id\":\"sb_demo\"}")
	secret := "demo-secret"

	mac := hmac.New(sha256.New, []byte(secret))
	mac.Write(payload)
	signature := "sha256=" + hex.EncodeToString(mac.Sum(nil))

	if !verifySignature(payload, signature, secret) {
		log.Fatal("signature verification failed")
	}
	fmt.Println("signature verified")
}`,
      mode: "run",
      needs: "none",
      session: "webhooks-signature-go"
    },
    {
      label: "Python",
      language: "python",
      code: `import hashlib
import hmac


def verify_signature(payload: bytes, signature: str, secret: str) -> bool:
    if not signature.startswith("sha256="):
        return False
    expected = signature[7:]
    mac = hmac.new(secret.encode(), payload, hashlib.sha256)
    return hmac.compare_digest(expected, mac.hexdigest())


payload = b'{"event":"process.exited","sandbox_id":"sb_demo"}'
secret = "demo-secret"
signature = "sha256=" + hmac.new(secret.encode(), payload, hashlib.sha256).hexdigest()

print("signature verified" if verify_signature(payload, signature, secret) else "invalid signature")`,
      mode: "run",
      needs: "none",
      session: "webhooks-signature-python"
    },
    {
      label: "JavaScript",
      language: "typescript",
      code: `import crypto from 'crypto';

function verifySignature(payload: string, signature: string, secret: string): boolean {
    if (!signature.startsWith('sha256=')) {
        return false;
    }
    const expected = signature.slice(7);
    const actual = crypto.createHmac('sha256', secret).update(payload).digest('hex');
    return crypto.timingSafeEqual(Buffer.from(expected, 'hex'), Buffer.from(actual, 'hex'));
}

const payload = JSON.stringify({ event: 'process.exited', sandbox_id: 'sb_demo' });
const secret = 'demo-secret';
const signature = 'sha256=' + crypto.createHmac('sha256', secret).update(payload).digest('hex');

console.log(verifySignature(payload, signature, secret) ? 'signature verified' : 'invalid signature');`,
      mode: "run",
      needs: "none",
      session: "webhooks-signature-typescript"
    },
    {
      label: "CLI",
      language: "bash",
      code: `PAYLOAD='{"event":"process.exited","sandbox_id":"sb_demo"}'
SECRET='demo-secret'
SIG="sha256=$(printf '%s' "$PAYLOAD" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $2}')"

CALC="$(printf '%s' "$PAYLOAD" | openssl dgst -sha256 -hmac "$SECRET" | awk '{print $2}')"
if [ "\${SIG#sha256=}" = "$CALC" ]; then
  echo "signature verified"
else
  echo "invalid signature"
  exit 1
fi`,
      mode: "run",
      needs: "none",
      session: "webhooks-signature-cli"
    }

## Best Practices

<Callout variant="success">
**Use a queue** - Webhook endpoints should respond quickly (within 5 seconds). Process events asynchronously using a message queue for reliability.
</Callout>

<Callout variant="warning">
**Verify signatures** - Always verify the webhook signature to ensure requests are from Sandbox0.
</Callout>

<Callout variant="info">
**Handle retries** - Sandbox0 retries failed webhook deliveries with exponential backoff. Make your endpoint idempotent.
</Callout>
