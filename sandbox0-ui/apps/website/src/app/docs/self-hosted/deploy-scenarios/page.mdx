# Deploy Scenarios

Sandbox0 supports multiple deployment scenarios to fit different use cases, from quick evaluation to production multi-cluster setups.

## Single-Cluster Scenarios

### Minimal Mode

<Badge variant="accent">Quick Start</Badge>

The simplest deployment with basic sandbox functionality.

**Components:** `internal-gateway` + `manager`

**Use Case:**
- Quick evaluation and testing
- Development environments
- Basic sandbox execution without persistence

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
  name: minimal
  namespace: sandbox0-system
spec:
  version: "latest"
  database:
    type: builtin
    builtin:
      enabled: true
      persistence:
        enabled: true
        size: "20Gi"
  publicExposure:
    enabled: true
    rootDomain: sandbox0.app
    regionId: aws-us-east-1
  services:
    internalGateway:
      enabled: true
      service:
        type: NodePort
        port: 30080
      config:
        authMode: public
    manager:
      enabled: true`,
      mode: "read",
      needs: "none",
      session: "selfhosted-scenarios-minimal-yaml"
    }
  ]}
/>

### Volumes Mode

<Badge variant="success">Production Ready</Badge>

Adds persistent volume support via storage-proxy.

**Components:** `internal-gateway` + `manager` + `storage-proxy`

**Use Case:**
- Applications requiring persistent data
- File system operations
- Volume snapshots and restoration

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
  name: volumes
  namespace: sandbox0-system
spec:
  version: "latest"
  database:
    type: builtin
    builtin:
      enabled: true
      persistence:
        size: "20Gi"
  storage:
    # Use builtin for development; switch to s3/oss for production
    type: builtin
    builtin:
      enabled: true
      persistence:
        size: "20Gi"
  publicExposure:
    enabled: true
    rootDomain: sandbox0.app
    regionId: aws-us-east-1
  services:
    internalGateway:
      enabled: true
      service:
        type: NodePort
        port: 30080
      config:
        authMode: public
    manager:
      enabled: true
    storageProxy:
      enabled: true
      config:
        juicefsEncryptionEnabled: false`,
      mode: "read",
      needs: "none",
      session: "selfhosted-scenarios-volumes-yaml"
    }
  ]}
/>

## Multi-Cluster Scenarios

Multi-cluster deployments separate control plane and data plane for horizontal scaling.

### Control Plane

Deploy the control plane to manage multiple data plane clusters.

**Components:** `edge-gateway` + `scheduler`

**Requirements:**
- External PostgreSQL (shared with data planes)
- Ingress for external access

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
  name: s0cp
  namespace: sandbox0-system
spec:
  version: "latest"
  database:
    # Control plane and data planes must share the same PostgreSQL
    type: external
    external:
      host: your-db.rds.amazonaws.com
      port: 5432
      database: sandbox0
      username: sandbox0
      passwordSecret:
        name: db-credentials
        key: password
  publicExposure:
    enabled: true
    rootDomain: sandbox0.app
    regionId: aws-us-east-1
  services:
    edgeGateway:
      enabled: true
      service:
        type: NodePort
        port: 30080
      config:
        schedulerEnabled: true
    scheduler:
      enabled: true`,
      mode: "read",
      needs: "none",
      session: "selfhosted-scenarios-control-plane-yaml"
    }
  ]}
/>

### Data Plane

Deploy data planes in different clusters/regions.

**Components:** `internal-gateway` + `manager` + `storage-proxy` + `netd`

**Requirements:**
- Same PostgreSQL as control plane
- Connection to control plane
- Unique cluster ID

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
  name: s0dp
  namespace: sandbox0-system
spec:
  version: "latest"
  database:
    # Must match control plane database
    type: external
    external:
      host: your-db.rds.amazonaws.com
      port: 5432
      database: sandbox0
      username: sandbox0
      passwordSecret:
        name: db-credentials
        key: password
  storage:
    # Each data plane has its own S3 bucket
    type: s3
    s3:
      bucket: sandbox0-prod-cluster1
      region: us-east-1
      endpoint: https://s3.amazonaws.com
      credentialsSecret:
        name: aws-credentials
  controlPlane:
    url: https://api.sandbox0.ai
    internalAuthPublicKeySecret:
      name: control-plane-public-key
      key: public.key
  cluster:
    id: cluster-001
    name: Production US East 1
  services:
    internalGateway:
      enabled: true
      service:
        type: NodePort
        port: 30080
      config:
        authMode: internal
    manager:
      enabled: true
    storageProxy:
      enabled: true
    netd:
      enabled: true`,
      mode: "read",
      needs: "none",
      session: "selfhosted-scenarios-data-plane-yaml"
    }
  ]}
/>

## Choosing a Scenario

| Scenario | Volumes | Network Policy | Multi-Cluster | Complexity |
|----------|---------|----------------|---------------|------------|
| Minimal | No | No | No | Low |
| Volumes | Yes | No | No | Medium |
| Network Policy | No | Yes | No | Medium |
| Full | Yes | Yes | No | Medium |
| Multi-cluster | Yes | Yes | Yes | High |

### Recommendations

<Callout variant="info">
**Development/Testing:** Start with Minimal mode, then add features as needed.

**Production:** Use Full mode with external PostgreSQL and S3 for reliability and scalability.

**High Scale:** Deploy multi-cluster with one control plane and multiple data planes across regions.
</Callout>
