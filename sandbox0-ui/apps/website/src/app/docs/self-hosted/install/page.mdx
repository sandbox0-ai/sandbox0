# Install

This is the shortest production-like path from zero to a running self-hosted Sandbox0.

## Prerequisites

- Kubernetes `1.35+`
- Helm `3.8+`
- `kubectl` matching your cluster minor version
- A default `StorageClass`

<Callout variant="warning">
`netd` and `storage-proxy` require Linux worker nodes. For local macOS/Windows testing, start with `minimal` mode.
</Callout>

## 1) Install infra-operator

```bash
helm repo add sandbox0 https://sandbox0-ai.github.io/sandbox0
helm repo update

helm install infra-operator sandbox0/infra-operator \
    --namespace sandbox0-system \
    --create-namespace
```

Verify operator + CRD:

```bash
kubectl get pods -n sandbox0-system
kubectl get crd sandbox0infras.infra.sandbox0.ai
```

## 2) Deploy Minimal Mode

Apply the official sample from source:

```bash
kubectl apply -f https://raw.githubusercontent.com/sandbox0-ai/sandbox0/main/infra-operator/chart/samples/single-cluster/minimal.yaml
```

Watch status:

```bash
kubectl get sandbox0infra -n sandbox0-system -w
```

Expected `status.phase` lifecycle:

1. `Installing`
2. `Upgrading` (when reconciling changes)
3. `Ready`

## 3) Verify Endpoint

```bash
kubectl get svc -n sandbox0-system -l app.kubernetes.io/component=internal-gateway
```

If using `NodePort` sample:

```bash
NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[0].address}')
NODE_PORT=$(kubectl get svc -n sandbox0-system -l app.kubernetes.io/component=internal-gateway -o jsonpath='{.items[0].spec.ports[0].nodePort}')

curl http://${NODE_IP}:${NODE_PORT}/healthz
```

## 4) (Optional) Initial Admin User

If `spec.initUser.passwordSecret` is not provided, operator generates `admin-password`.

```bash
kubectl get secret admin-password -n sandbox0-system -o jsonpath='{.data.password}' | base64 -d
```

## Next

- Need persistent volume + snapshots: enable `storageProxy` and `storage` backend in your `Sandbox0Infra`
- Need network policy enforcement: enable `services.netd`
- Need multi-cluster: use `multi-cluster/control-plane.yaml` + `multi-cluster/data-plane.yaml` as your baseline
