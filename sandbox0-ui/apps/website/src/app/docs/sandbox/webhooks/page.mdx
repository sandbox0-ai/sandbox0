# Webhooks

Receive real-time event notifications from sandboxes via webhooks. Webhooks allow you to react to process events, sandbox lifecycle changes, and file modifications.

## Webhook Configuration

Configure webhooks when claiming a sandbox:

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Webhook endpoint URL (required) |
| `secret` | string | Shared secret for payload signing (optional) |
| `watch_dir` | string | Directory to monitor for file events (optional) |

<Callout variant="info">
Webhooks are configured per-sandbox at creation time. Each sandbox can have its own webhook configuration.
</Callout>

<Callout variant="success">
`secret` is optional. For quick debugging, you can leave it empty and use services like <a href="https://webhook.site" target="_blank" rel="noopener noreferrer" className="text-blue-600 decoration-blue-500 underline-offset-2 hover:text-blue-500 dark:text-blue-400 dark:decoration-blue-400">webhook.site</a> to inspect incoming events directly.
</Callout>

## Event Types

### Sandbox Events

| Event | Description |
|-------|-------------|
| `sandbox.ready` | Sandbox has been initialized and is ready |
| `sandbox.paused` | Sandbox has been paused |
| `sandbox.resumed` | Sandbox has been resumed |
| `sandbox.killed` | Sandbox has been killed |

### Process Events

| Event | Description |
|-------|-------------|
| `process.started` | A new process/context has started |
| `process.exited` | A process has exited (includes exit code) |
| `process.crashed` | A process has crashed unexpectedly |

### File Events

| Event | Description |
|-------|-------------|
| `file.modified` | A file event occurred (create, write, remove, rename, or chmod) |

<Callout variant="info">
File events require setting <code>watch_dir</code> in the webhook configuration. The specific operation type is included in the payload's <code>event_type</code> field.
</Callout>

### Agent Events

| Event | Description |
|-------|-------------|
| `agent.event` | Custom event published by the sandbox via the webhook publish API |

## Signature Verification

When a secret is configured, webhook payloads are signed using HMAC-SHA256.

<Callout variant="info">
If `secret` is not configured, Sandbox0 will not send the `X-Sandbox0-Signature` header.
</Callout>

### Header

```
X-Sandbox0-Signature: <hex-encoded-signature>
```

The signature is the hex-encoded HMAC-SHA256 of the request body.

### Verification Example

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `import (
    "io"
    "net/http"
    "os"

    sandbox0 "github.com/sandbox0-ai/sdk-go"
)

func webhookHandler(w http.ResponseWriter, r *http.Request) {
    // Verify against raw request bytes, not a re-serialized JSON object.
    payload, err := io.ReadAll(r.Body)
    if err != nil {
        http.Error(w, "read body failed", http.StatusBadRequest)
        return
    }
    signature := r.Header.Get("X-Sandbox0-Signature")
    secret := os.Getenv("SANDBOX0_WEBHOOK_SECRET")

    if !sandbox0.VerifyWebhookSignature(secret, payload, signature) {
        http.Error(w, "invalid signature", http.StatusUnauthorized)
        return
    }

    // Process webhook...
    w.WriteHeader(http.StatusOK)
}`
    },
    {
      label: "Python",
      language: "python",
      code: `import os
from flask import Flask, request
from sandbox0 import verify_webhook_signature

app = Flask(__name__)

@app.route('/webhook', methods=['POST'])
def webhook():
    # Verify against raw request bytes, not a re-serialized JSON object.
    payload = request.get_data()
    signature = request.headers.get('X-Sandbox0-Signature', '')
    secret = os.environ.get('SANDBOX0_WEBHOOK_SECRET', '')

    if not verify_webhook_signature(secret, payload, signature):
        return 'Invalid signature', 401

    event = request.json
    print(f"Received event: {event['event_type']}")
    return 'OK', 200`
    },
    {
      label: "TypeScript",
      language: "typescript",
      code: `import { verifyWebhookSignature } from 'sandbox0';
import express from 'express';

const app = express();

app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
    // Verify against raw request bytes, not a re-serialized JSON object.
    const payload = req.body as Buffer;
    const signature = (req.headers['x-sandbox0-signature'] as string) || '';
    const secret = process.env.SANDBOX0_WEBHOOK_SECRET || '';

    if (!verifyWebhookSignature(secret, payload, signature)) {
        res.status(401).send('Invalid signature');
        return;
    }

    const event = JSON.parse(payload.toString('utf8'));
    console.log('Received event:', event.event_type);
    res.status(200).send('OK');
});`
    },
    {
      label: "CLI",
      language: "bash",
      code: `# Webhook server example using netcat (for testing)
# In production, use a proper HTTP server

while true; do
    echo -e "HTTP/1.1 200 OK\\r\\n\\r\\nOK" | nc -l -p 8080 -q 1
done`
    }
