  1. PostgreSQL and RustFS (infrastructure)

     - CRDs (sandboxtemplates)
     - Manager deployment
  3. Default SandboxTemplate CR '{{ .Values.defaultTemplate.name }}' (post-install hook, weight 10)
  4. Other services (storage-proxy, netd, internal-gateway, edge-gateway)

NOTE: Network and bandwidth policies are stored as Pod annotations instead of CRDs.

VERIFY INSTALLATION:
  # Check if namespace was created
  $ kubectl get namespace {{ .Values.manager.config.default_template_namespace }}

  # Check if CRDs are installed
  $ kubectl get crd | grep sandbox0.ai

  # Check if default template is created
  $ kubectl get sandboxtemplate -n {{ .Values.manager.config.default_template_namespace }}

  # Check all pods are running
  $ kubectl get pods -n {{ .Release.Namespace }}

IMPORTANT: Before deploying to production, make sure to:

1. Create the internal JWT secret with private/public key pair:

   kubectl -n sandbox0-system create secret generic {{ .Values.global.jwt.internalJwtSecretName }} \
     --from-file={{ .Values.global.jwt.privateKeyKey }}=/path/to/private.key \
     --from-file={{ .Values.global.jwt.publicKeyKey }}=/path/to/public.key

2. Update the following values in your values.yaml or via --set:

   - global.database.url
   - global.juicefs.meta_url
   - global.juicefs.s3_bucket
   - global.juicefs.s3_access_key
   - global.juicefs.s3_secret_key
   - global.jwt.secret

3. If using built-in authentication, configure:

   - global.initUser.email
   - global.initUser.password
   - global.initUser.name
