# Volume

{/* TODO(volume-page): align examples with latest generated SDK signatures */}

Volume provides persistent storage for Sandbox0. A volume is independent from sandbox lifecycle, so data can outlive a single runtime and be shared across workloads.

## Why Volumes?

- **Persistence**: Keep data after sandbox deletion
- **Sharing**: Reuse one volume across multiple sandboxes
- **Snapshots**: Capture point-in-time state for backup and rollback
- **Recovery**: Restore quickly when deployment or data changes fail

## Volume and Sandbox Relationship

```
┌─────────────────────────────────────────────────────┐
│                      Volume                         │
│                 (Persistent Storage)                │
│                                                     │
│    /data/                                           │
│    ├── models/                                      │
│    ├── cache/                                       │
│    └── output/                                      │
└──────────┬──────────────┬──────────────┬───────────┘
           │              │              │
           ▼              ▼              ▼
     ┌──────────┐   ┌──────────┐   ┌──────────┐
     │ Sandbox A│   │ Sandbox B│   │ Sandbox C│
     │ (RWX)    │   │ (ROX)    │   │ (RWX)    │
     └──────────┘   └──────────┘   └──────────┘
```

## Access Modes

| Mode | Full Name | Description | Typical Use Cases |
|------|-----------|-------------|-------------------|
| `RWO` | Read-Write Once | Single-node read-write | Database storage, exclusive workspaces |
| `ROX` | Read-Only Cross | Multi-node read-only | Shared model files, static resources |
| `RWX` | Read-Write Cross | Multi-node read-write | Shared caches, collaborative workloads |

<Callout variant="info">
Default mode is `RWO`. Use `RWX` when a volume must be mounted by multiple sandboxes.
</Callout>

## Performance Configuration

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `cache_size` | string | Local read cache size | `512M`, `1G` |
| `buffer_size` | string | Write buffer size | `128M`, `256M` |
| `prefetch` | integer | Prefetch blocks | `1`, `2`, `4` |
| `writeback` | boolean | Async writeback mode | `true`, `false` |

<Callout variant="warning">
`writeback=true` improves throughput but may lose buffered data on abnormal shutdown.
</Callout>

## Create a Volume

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `import (
    "context"
    "fmt"
    "log"
    "os"

    sandbox0 "github.com/sandbox0-ai/sdk-go"
    "github.com/sandbox0-ai/sdk-go/pkg/apispec"
)

ctx := context.Background()
client, err := sandbox0.NewClient(
    sandbox0.WithToken(os.Getenv("SANDBOX0_TOKEN")),
)
if err != nil {
    log.Fatal(err)
}

volume, err := client.CreateVolume(ctx, apispec.CreateSandboxVolumeRequest{
    AccessMode: apispec.NewOptVolumeAccessMode(apispec.VolumeAccessModeRWX),
    CacheSize:  apispec.NewOptString("512M"),
    BufferSize: apispec.NewOptString("128M"),
    Writeback:  apispec.NewOptBool(true),
})
if err != nil {
    log.Fatal(err)
}
fmt.Println("Volume created:", volume.ID)`,
      mode: "run",
      needs: "infra",
      session: "volume-create-go",
      deps: ["github.com/sandbox0-ai/sdk-go"],
      env: ["SANDBOX0_TOKEN"],
      setup: ["go version"]
    },
    {
      label: "Python",
      language: "python",
      code: `import os

from sandbox0 import Client
from sandbox0.models import CreateSandboxVolumeRequest, VolumeAccessMode

client = Client(token=os.environ["SANDBOX0_TOKEN"])

volume = client.volumes.create(
    CreateSandboxVolumeRequest(
        access_mode=VolumeAccessMode.RWX,
        cache_size="512M",
        buffer_size="128M",
        writeback=True,
    )
)
print("Volume created:", volume.id)`,
      mode: "run",
      needs: "infra",
      session: "volume-create-python",
      deps: ["sandbox0"],
      env: ["SANDBOX0_TOKEN"],
      setup: ["python3 --version"]
    },
    {
      label: "JavaScript",
      language: "typescript",
      code: `import { Sandbox0 } from '@sandbox0/sdk';
import { VolumeAccessMode } from '@sandbox0/sdk/models';

if (!process.env.SANDBOX0_TOKEN) {
    throw new Error('Missing SANDBOX0_TOKEN');
}

const client = new Sandbox0({ token: process.env.SANDBOX0_TOKEN });

const volume = await client.volumes.create({
    accessMode: VolumeAccessMode.Rwx,
    cacheSize: '512M',
    bufferSize: '128M',
    writeback: true,
});
console.log('Volume created:', volume.id);`,
      mode: "run",
      needs: "infra",
      session: "volume-create-typescript",
      deps: ["@sandbox0/sdk"],
      env: ["SANDBOX0_TOKEN"],
      setup: ["node --version"]
    },
    {
      label: "CLI",
      language: "bash",
      code: `s0 volume create --access-mode RWX --cache-size 512M --buffer-size 128M --writeback`,
      mode: "run",
      needs: "infra",
      session: "volume-create-cli",
      setup: ["command -v s0 >/dev/null"]
    }
  ]}
