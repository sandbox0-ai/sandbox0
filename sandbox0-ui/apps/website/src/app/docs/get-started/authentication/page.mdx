# Authentication

Sandbox0 supports multiple authentication methods to secure your API access. This guide covers all available options and best practices.

## Authentication Methods

Sandbox0 provides two primary authentication methods:

| Method | Use Case | Description |
|--------|----------|-------------|
| **Access Token** | User sessions | JWT token obtained via login (email/password or OIDC) |
| **API Key** | Server-to-server | Long-lived key for automated systems and services |

---

## Access Token Authentication

Access tokens are JWT tokens obtained through the authentication flow. They have a limited lifetime and can be refreshed.

### Login with Email/Password

<Endpoint method="POST">
/auth/login
</Endpoint>

#### Request Body

| Field | Type | Description |
|-------|------|-------------|
| `email` | string | User email address |
| `password` | string | User password |

<Tabs

### Token Refresh

Access tokens expire after a configured time. Use the refresh token to obtain a new access token.

<Endpoint method="POST">
/auth/refresh
</Endpoint>

#### Request Body

| Field | Type | Description |
|-------|------|-------------|
| `refresh_token` | string | The refresh token from login |

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `// Refresh the access token
resp, err := client.RefreshToken(ctx, &sandbox0.RefreshRequest{
    RefreshToken: storedRefreshToken,
})
if err != nil {
    log.Fatal(err)
}

// Update stored tokens
fmt.Printf("New access token: %s\\n", resp.AccessToken)`
    },
    {
      label: "Python",
      language: "python",
      code: `# Refresh the access token
resp = client.refresh_token(refresh_token=stored_refresh_token)
print(f"New access token: {resp.access_token}")`
    },
    {
      label: "JavaScript",
      language: "typescript",
      code: `// Refresh the access token
const resp = await client.refreshToken({
    refreshToken: storedRefreshToken,
});
console.log('New access token:', resp.accessToken);`
    },
    {
      label: "CLI",
      language: "bash",
      code: `# Refresh tokens (handled automatically by CLI)
s0 auth refresh`
    }
  ]}
/>

<Callout variant="info">
SDKs handle token refresh automatically. When using the raw API, implement refresh logic to maintain sessions.
</Callout>

### Logout

Invalidate the current session.

<Endpoint method="POST">
/auth/logout
</Endpoint>

## API Key Authentication

API keys are long-lived credentials designed for server-to-server communication and automated systems.

### Creating API Keys

API keys are created through the dashboard or API after authentication.

<Endpoint method="POST">
/api-keys
</Endpoint>

<Callout variant="warning">
API keys should be treated as secrets. Never expose them in client-side code or public repositories.
</Callout>

### Using API Keys

Pass the API key in the `Authorization` header as a Bearer token:

<Tabs

### Managing API Keys

| Operation | Endpoint | Description |
|-----------|----------|-------------|
| List keys | `GET /api-keys` | List all API keys for the current user |
| Create key | `POST /api-keys` | Create a new API key |
| Deactivate | `POST /api-keys/{'{id}'}/deactivate` | Temporarily disable a key |
| Delete | `DELETE /api-keys/{'{id}'}` | Permanently delete a key |

## Best Practices

### Token Storage

<Callout variant="warning">
Never store tokens or API keys in source code or version control.
</Callout>

Recommended approaches:

1. **Environment Variables**: Use environment variables for API keys and tokens
2. **Secret Managers**: Use tools like AWS Secrets Manager, HashiCorp Vault
3. **Secure Config Files**: Use file permissions to restrict access to config files

### Token Rotation

- Rotate API keys periodically (recommended: every 90 days)
- Use refresh tokens to obtain new access tokens before expiration
- Deactivate compromised keys immediately

### Scope and Permissions

- Create separate API keys for different services/environments
- Use the principle of least privilege when assigning team roles
- Monitor API key usage through audit logs

## Error Handling

Authentication errors return standard HTTP status codes:

| Status | Description |
|--------|-------------|
| `401` | Invalid or expired token |
| `403` | Insufficient permissions |
| `429` | Rate limit exceeded |

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `sandbox, err := client.ClaimSandbox(ctx, "default")
if err != nil {
    if errors.Is(err, sandbox0.ErrUnauthorized) {
        // Token expired or invalid - re-authenticate
        log.Println("Token expired, please re-authenticate")
    } else if errors.Is(err, sandbox0.ErrForbidden) {
        // Insufficient permissions
        log.Println("Insufficient permissions for this operation")
    }
    log.Fatal(err)
}`
    },
    {
      label: "Python",

    sandbox = client.claim_sandbox(template="default")
except errors.UnauthorizedError:

    print("Token expired, please re-authenticate")
except errors.ForbiddenError:
    # Insufficient permissions
    print("Insufficient permissions for this operation")`
    },
    {
      label: "JavaScript",
      language: "typescript",

    if (error instanceof UnauthorizedError) {
        // Token expired or invalid
