# Volume Mounts

After mounting a Volume to a Sandbox, you can access Volume data within the Sandbox just like a local filesystem.

## Mount Flow

```
┌─────────────┐      Mount Request      ┌─────────────┐
│   Sandbox   │ ──────────────────────► │    Volume   │
│             │                         │             │
│ /mnt/data ◄─┼─────── Mounted ─────────┤   /data     │
└─────────────┘                         └─────────────┘
```

## Mount a Volume

When mounting, you need to specify:
- `volume_id`: The Volume ID to mount
- `mount_point`: The mount path inside the Sandbox
- `volume_config` (optional): Performance configuration override at mount time

## Share Volume Across Sandboxes

Volumes with `RWX` access mode can be mounted to multiple Sandboxes simultaneously:

<Tabs
  tabs={[
    { label: "Go", language: "go", code: "// Create RWX Volume\nvolume, _ := client.CreateVolume(ctx, apispec.CreateSandboxVolumeRequest{\n    AccessMode: apispec.NewOptVolumeAccessMode(apispec.VolumeAccessModeRWX),\n})\n\n// Create two Sandboxes\nsandbox1, _ := client.ClaimSandbox(ctx, apispec.ClaimSandboxRequest{})\nsandbox2, _ := client.ClaimSandbox(ctx, apispec.ClaimSandboxRequest{})\n\n// Mount same Volume to both Sandboxes\nmount1, _ := client.MountVolume(ctx, sandbox1.ID, apispec.MountRequest{\n    SandboxvolumeID: volume.ID,\n    MountPoint:      \"/mnt/shared\",\n})\n\nmount2, _ := client.MountVolume(ctx, sandbox2.ID, apispec.MountRequest{\n    SandboxvolumeID: volume.ID,\n    MountPoint:      \"/mnt/shared\",\n})\n\n// Sandbox1 writes file\nclient.WriteFile(ctx, sandbox1.ID, apispec.WriteFileRequest{\n    Path:    \"/mnt/shared/message.txt\",\n    Content: []byte(\"Hello from Sandbox1\"),\n})\n\n// Sandbox2 can read it\ncontent, _ := client.ReadFile(ctx, sandbox2.ID, \"/mnt/shared/message.txt\")\nfmt.Printf(\"Sandbox2 read: %s\\n\", string(content))\n// Output: Sandbox2 read: Hello from Sandbox1" },
    { label: "Python", language: "python", code: "from sandbox0.models import CreateSandboxVolumeRequest, VolumeAccessMode\n\n# Create RWX Volume\nvolume = client.volumes.create(CreateSandboxVolumeRequest(\n    access_mode=VolumeAccessMode.RWX,\n))\n\n# Create two Sandboxes\nsandbox1 = client.sandboxes.claim(ClaimSandboxRequest())\nsandbox2 = client.sandboxes.claim(ClaimSandboxRequest())\n\n# Mount same Volume to both Sandboxes\nwith sandbox1.mount(volume.id, \"/mnt/shared\"):\n    with sandbox2.mount(volume.id, \"/mnt/shared\"):\n        # Sandbox1 writes file\n        sandbox1.write_file(\"/mnt/shared/message.txt\", b\"Hello from Sandbox1\")\n        \n        # Sandbox2 can read it\n        content = sandbox2.read_file(\"/mnt/shared/message.txt\")\n        print(f\"Sandbox2 read: {content.decode()}\")\n        # Output: Sandbox2 read: Hello from Sandbox1" },
    { label: "TypeScript", language: "typescript", code: "import { VolumeAccessMode } from 'sandbox0/models';\n\n// Create RWX Volume\nconst volume = await client.volumes.create({\n    accessMode: VolumeAccessMode.Rwx,\n});\n\n// Create two Sandboxes\nconst sandbox1 = await client.sandboxes.claim({});\nconst sandbox2 = await client.sandboxes.claim({});\n\n// Mount same Volume to both Sandboxes\nconst mount1 = await client.sandboxes.mount(sandbox1.id, {\n    sandboxvolumeId: volume.id,\n    mountPoint: '/mnt/shared',\n});\n\nconst mount2 = await client.sandboxes.mount(sandbox2.id, {\n    sandboxvolumeId: volume.id,\n    mountPoint: '/mnt/shared',\n});\n\n// Sandbox1 writes file\nawait client.sandboxes.writeFile(sandbox1.id, {\n    path: '/mnt/shared/message.txt',\n    content: Buffer.from('Hello from Sandbox1'),\n});\n\n// Sandbox2 can read it\nconst content = await client.sandboxes.readFile(sandbox2.id, '/mnt/shared/message.txt');\nconsole.log(`Sandbox2 read: ${content.toString()}`);\n// Output: Sandbox2 read: Hello from Sandbox1\n\n// Cleanup\nawait client.sandboxes.unmount(sandbox1.id, { sandboxvolumeId: volume.id, mountSessionId: mount1.mountSessionId });\nawait client.sandboxes.unmount(sandbox2.id, { sandboxvolumeId: volume.id, mountSessionId: mount2.mountSessionId });" },
    { label: "CLI", language: "bash", code: "# Create RWX Volume\ns0 volume create --access-mode RWX\n# Output: Volume created: vol_shared123\n\n# Create two Sandboxes\ns0 sandbox create --template tpl_default\n# Output: Sandbox created: sb_one111\ns0 sandbox create --template tpl_default\n# Output: Sandbox created: sb_two222\n\n# Mount to both Sandboxes\ns0 sandbox mount sb_one111 --volume vol_shared123 --path /mnt/shared\ns0 sandbox mount sb_two222 --volume vol_shared123 --path /mnt/shared\n\n# Sandbox1 writes file\ns0 sandbox exec sb_one111 -- sh -c 'echo \"Hello from Sandbox1\" > /mnt/shared/message.txt'\n\n# Sandbox2 reads file\ns0 sandbox exec sb_two222 -- cat /mnt/shared/message.txt\n# Output: Hello from Sandbox1" }
  ]}
/>

<Callout variant="info">
With `RWX` mode, multiple Sandboxes can read and write to the same Volume simultaneously. Handle concurrent write conflicts appropriately.
</Callout>

## Troubleshooting

### Mount Fails

<Callout variant="warning">
If mount fails, check:
1. Volume exists and is not deleted
2. Sandbox is in `running` state
3. Mount path is not already in use
4. Access mode matches use case (e.g., multiple mounts on `RWO` mode)
</Callout>

### 409 Conflict When Deleting Volume

<Callout variant="danger">
Cannot delete a Volume with active mounts. Unmount all mounts first before deleting.
</Callout>
