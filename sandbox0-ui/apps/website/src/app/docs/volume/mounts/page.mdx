# Volume Mounts

{/* TODO(volume-mounts-page): add explicit mount conflict examples for RWO */}

After a volume is mounted, the mount path inside sandbox behaves like a local POSIX directory.

## Mount Flow

```
┌─────────────┐      Mount Request      ┌─────────────┐
│   Sandbox   │ ──────────────────────► │    Volume   │
│             │                         │             │
│ /mnt/data ◄─┼─────── Mounted ─────────┤   /data     │
└─────────────┘                         └─────────────┘
```

## Mount a Volume

- `volume_id`: volume to mount
- `mount_point`: target path in sandbox
- `volume_config`: optional runtime override for cache and buffering

<Tabs

      label: "Go",
      language: "go",
      code: `import (
    "context"
    "fmt"
    "log"
    "os"

    "github.com/sandbox0-ai/sdk-go/pkg/apispec"
)

ctx := context.Background()
client, err := sandbox0.NewClient(
    sandbox0.WithToken(os.Getenv("SANDBOX0_TOKEN")),
)
if err != nil {
    log.Fatal(err)
}

sandbox, err := client.ClaimSandbox(ctx, apispec.ClaimSandboxRequest{})
if err != nil {
    log.Fatal(err)
}

volume, err := client.CreateVolume(ctx, apispec.CreateSandboxVolumeRequest{
    AccessMode: apispec.NewOptVolumeAccessMode(apispec.VolumeAccessModeRWX),
})
if err != nil {
    log.Fatal(err)
}

mount, err := client.MountVolume(ctx, sandbox.ID, apispec.MountRequest{
    SandboxvolumeID: volume.ID,
    MountPoint: "/mnt/data",
})
if err != nil {
    log.Fatal(err)
}
fmt.Println("Mounted at:", mount.MountPoint)`,
      mode: "run",
      needs: "infra",
      session: "volume-mount-go",
      deps: ["github.com/sandbox0-ai/sdk-go"],
      env: ["SANDBOX0_TOKEN"],
      setup: ["go version"]
    },
    {
      label: "Python",
      language: "python",
      code: `import os

from sandbox0 import Client
from sandbox0.models import ClaimSandboxRequest, CreateSandboxVolumeRequest, VolumeAccessMode

client = Client(token=os.environ["SANDBOX0_TOKEN"])

sandbox = client.sandboxes.claim(ClaimSandboxRequest())
volume = client.volumes.create(
    CreateSandboxVolumeRequest(access_mode=VolumeAccessMode.RWX)
)

mount_session = sandbox.mount(volume.id, "/mnt/data")
print("Mounted at:", mount_session.mount_point)`,
      mode: "run",
      needs: "infra",
      session: "volume-mount-python",
      deps: ["sandbox0"],
      env: ["SANDBOX0_TOKEN"],
      setup: ["python3 --version"]

    {
      label: "JavaScript",
      language: "typescript",
      code: `import { Sandbox0 } from '@sandbox0/sdk';
import { VolumeAccessMode } from '@sandbox0/sdk/models';

if (!process.env.SANDBOX0_TOKEN) {
    throw new Error('Missing SANDBOX0_TOKEN');
}

const client = new Sandbox0({ token: process.env.SANDBOX0_TOKEN });

const sandbox = await client.sandboxes.claim({});
const volume = await client.volumes.create({
    accessMode: VolumeAccessMode.Rwx,
});

const mountSession = await client.sandboxes.mount(sandbox.id, {
    sandboxvolumeId: volume.id,
    mountPoint: '/mnt/data',

console.log('Mounted at:', mountSession.mountPoint);`,

      setup: ["node --version"]

## Share Volume Across Sandboxes

`RWX` volumes can be mounted by multiple sandboxes at the same time.

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `volume, err := client.CreateVolume(ctx, apispec.CreateSandboxVolumeRequest{
    AccessMode: apispec.NewOptVolumeAccessMode(apispec.VolumeAccessModeRWX),
})
if err != nil {
    log.Fatal(err)
}

sandboxA, _ := client.ClaimSandbox(ctx, apispec.ClaimSandboxRequest{})
sandboxB, _ := client.ClaimSandbox(ctx, apispec.ClaimSandboxRequest{})

_, _ = client.MountVolume(ctx, sandboxA.ID, apispec.MountRequest{
    SandboxvolumeID: volume.ID,
    MountPoint: "/mnt/shared",
})
_, _ = client.MountVolume(ctx, sandboxB.ID, apispec.MountRequest{
    SandboxvolumeID: volume.ID,
    MountPoint: "/mnt/shared",
})`,
      mode: "run",
      needs: "infra",
      session: "volume-share-go",
      env: ["SANDBOX0_TOKEN"]
    },
    {
      label: "Python",
      language: "python",
      code: `from sandbox0.models import ClaimSandboxRequest, CreateSandboxVolumeRequest, VolumeAccessMode

volume = client.volumes.create(
    CreateSandboxVolumeRequest(access_mode=VolumeAccessMode.RWX)
)
sandbox_a = client.sandboxes.claim(ClaimSandboxRequest())
sandbox_b = client.sandboxes.claim(ClaimSandboxRequest())

sandbox_a.mount(volume.id, "/mnt/shared")
sandbox_b.mount(volume.id, "/mnt/shared")`,
      mode: "run",
      needs: "infra",
      session: "volume-share-python",
      env: ["SANDBOX0_TOKEN"]
    },
    {
      label: "JavaScript",
      language: "typescript",
      code: `import { VolumeAccessMode } from '@sandbox0/sdk/models';

const sharedVolume = await client.volumes.create({
    accessMode: VolumeAccessMode.Rwx,
});
const sandboxA = await client.sandboxes.claim({});
const sandboxB = await client.sandboxes.claim({});

await client.sandboxes.mount(sandboxA.id, {
    sandboxvolumeId: sharedVolume.id,
    mountPoint: '/mnt/shared',
});
await client.sandboxes.mount(sandboxB.id, {
    sandboxvolumeId: sharedVolume.id,
    mountPoint: '/mnt/shared',
});`,
      mode: "run",
      needs: "infra",
      session: "volume-share-typescript",
      env: ["SANDBOX0_TOKEN"]
    },
    {
      label: "CLI",
      language: "bash",
      code: `s0 sandbox mount "$SANDBOX_A" --volume "$VOLUME_ID" --path /mnt/shared
s0 sandbox mount "$SANDBOX_B" --volume "$VOLUME_ID" --path /mnt/shared`,
      mode: "run",
      needs: "infra",
      session: "volume-share-cli",
      setup: ["command -v s0 >/dev/null"]
    }
  ]}
/>

<Callout variant="info">
With `RWX`, concurrent writes are allowed. Application-level conflict control is still required.
</Callout>

## Troubleshooting

### Mount Fails

<Callout variant="warning">
Check volume existence, sandbox running status, mount path conflict, and access mode compatibility.
</Callout>

### 409 Conflict on Delete

<Callout variant="danger">
A mounted volume cannot be deleted until all active mount sessions are unmounted.
</Callout>
