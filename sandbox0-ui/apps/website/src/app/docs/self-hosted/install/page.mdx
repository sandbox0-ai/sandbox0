# Install Infra Operator

This guide walks you through installing the sandbox0 infra-operator and deploying your first instance.

## Prerequisites

Before installing, ensure you have:

| Requirement | Version | Notes |
|-------------|---------|-------|
| Kubernetes | 1.23+ | Tested on 1.28+ |
| Helm | 3.8+ | For chart installation |
| kubectl | 1.23+ | For cluster interaction |
| StorageClass | - | Default storage class for PVCs |

### Kubernetes Requirements

<Callout variant="info">
For production deployments, you'll also need:
- **PostgreSQL**: External managed database (RDS, Cloud SQL, etc.)
- **Object Storage**: S3-compatible storage or Aliyun OSS
- **Container Registry**: ECR, GCR, ACR, or built-in registry
</Callout>

### Verify Prerequisites

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Check Kubernetes version
kubectl version

# Check Helm version
helm version

# Verify default storage class
kubectl get storageclass`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-prereq",

    }
  ]}

## Install the Operator

### Step 1: Add the Helm Repository

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `helm repo add sandbox0 https://charts.sandbox0.ai
helm repo update`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-helm-repo",
      setup: ["command -v helm >/dev/null"]
    }
  ]}
/>

### Step 2: Install the Operator

Install the infra-operator in the `sandbox0-system` namespace:

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `helm install infra-operator sandbox0/infra-operator \
  --namespace sandbox0-system \
  --create-namespace`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-operator",
      setup: ["command -v helm >/dev/null"]
    }
  ]}
/>

### Step 3: Verify Installation

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Check operator pod status
kubectl get pods -n sandbox0-system

# Check CRD installation
kubectl get crd sandbox0infras.infra.sandbox0.io`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-verify-operator",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>

Expected output:

```
NAME                        READY   STATUS    RESTARTS   AGE
infra-operator-xxx-xxx      1/1     Running   0          30s
```

## Deploy Your First Instance

### Quick Start: Minimal Mode

Create a file named `sandbox0infra-minimal.yaml`:

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `apiVersion: v1
kind: Namespace
metadata:
  name: sandbox0-system
---
apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
  name: minimal
  namespace: sandbox0-system
spec:
  version: "latest"
  database:
    type: builtin
    builtin:
      enabled: true
      persistence:
        enabled: true
        size: "20Gi"
  publicExposure:
    enabled: true
    rootDomain: sandbox0.app
    regionId: aws-us-east-1
  builtinTemplates:
    - templateId: default
      image: sandbox0ai/otemplates:default-v0.1.0
      displayName: Default
      description: Builtin template installed by infra-operator.
      pool:
        minIdle: 1
        maxIdle: 5
        autoScale: true
  services:
    internalGateway:
      enabled: true
      replicas: 1
      service:
        type: NodePort
        port: 30080
      config:
        authMode: public
    manager:
      enabled: true
      replicas: 1
  initUser:
    email: "admin@example.com"`,
      mode: "read",
      needs: "none",
      session: "selfhosted-install-minimal-yaml"
    }
  ]}
/>

Apply the configuration:

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `kubectl apply -f sandbox0infra-minimal.yaml`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-apply-minimal",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>

### Monitor Installation Progress

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Watch the Sandbox0Infra status
kubectl get sandbox0infra -n sandbox0-system -w

# Check detailed status
kubectl describe sandbox0infra minimal -n sandbox0-system`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-monitor",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>

The `Phase` field will transition through:

1. `Installing` - Components are being deployed
2. `Ready` - All components are healthy
3. `Degraded` - Some components are unhealthy
4. `Failed` - Installation failed

### Verify Deployment

Once `Phase` is `Ready`:

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Check all pods
kubectl get pods -n sandbox0-system

# Check services
kubectl get svc -n sandbox0-system

# Get the API endpoint (NodePort)
kubectl get svc -n sandbox0-system -l app.kubernetes.io/component=internal-gateway`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-verify-deploy",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>

## Access the API

### Get the Endpoint

For NodePort service:

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Get node IP
NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[0].address}')

# Get node port
NODE_PORT=$(kubectl get svc -n sandbox0-system -l app.kubernetes.io/component=internal-gateway -o jsonpath='{.items[0].spec.ports[0].nodePort}')

echo "API endpoint: http://${'$'}{NODE_IP}:${'$'}{NODE_PORT}"`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-get-endpoint",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>

## Initial User

The `initUser` configuration creates an initial admin user:

<Tabs
  tabs={[
    {
      label: "YAML",
      language: "yaml",
      code: `initUser:
  email: "admin@example.com"
  # Optional: reference a secret for password
  # passwordSecret:
  #   name: admin-password
  #   key: password`,
      mode: "read",
      needs: "none",
      session: "selfhosted-install-inituser-yaml"
    }
  ]}
/>

If `passwordSecret` is not specified, a random password is generated and stored in a secret:

<Tabs
  tabs={[
    {
      label: "CLI",
      language: "bash",
      code: `# Get the generated password
kubectl get secret -n sandbox0-system -l app.kubernetes.io/component=init-user -o jsonpath='{.items[0].data.password}' | base64 -d`,
      mode: "run",
      needs: "k8s",
      session: "selfhosted-install-init-user-password",
      setup: ["command -v kubectl >/dev/null"]
    }
  ]}
/>
