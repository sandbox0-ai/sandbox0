# Volume Snapshots

{/* TODO(volume-snapshots-page): add retention policy and lifecycle automation examples */}

A snapshot is a point-in-time read-only copy of a volume. Use snapshots for backup, rollback, and reproducible environment state.

## Snapshot Features

- **Instant Creation**: Copy-on-write snapshot creation is fast
- **Space Efficient**: Only changed blocks consume extra storage
- **Fast Recovery**: Restore volume state quickly
- **Version Tracking**: Use names and descriptions for release checkpoints

## Snapshot Lifecycle

```
┌─────────────────────────────────────────────────────────┐
│                      Volume                             │
│                                                         │
│    Snapshot 1          Snapshot 2          Snapshot 3  │
│    (v1.0-release)      (v1.1-beta)         (v2.0-dev)   │
│         │                   │                   │       │
│         ▼                   ▼                   ▼       │
│    ┌─────────┐         ┌─────────┐         ┌─────────┐ │
│    │ Snap A  │         │ Snap B  │         │ Snap C  │ │
│    └─────────┘         └─────────┘         └─────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
                          │
                          │ Restore
                          ▼
                   Restore to specific snapshot
```

## Restore a Snapshot

Restoring a snapshot reverts volume data to the selected checkpoint.

<Callout variant="warning">
Restore is irreversible for current state changes made after that snapshot point.
</Callout>

## Best Practices

### Naming

<Callout variant="info">
Use release-aligned names such as `v1.0.0`, `before-migration`, `post-import-check`.
</Callout>

### Cleanup

<Callout variant="warning">
Snapshots consume storage over time. Define retention policy by age, count, and business milestones.
</Callout>

### Backup Before Restore

<Callout variant="danger">
Create a fresh safety snapshot before any production restore operation.
</Callout>
