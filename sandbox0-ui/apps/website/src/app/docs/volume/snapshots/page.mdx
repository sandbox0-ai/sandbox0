# Volume Snapshots

A Snapshot is a point-in-time, read-only copy of a Volume. Snapshots are designed for backup, rollback, and reproducible environments.

## Snapshot Features

- **Instant Creation**: Built on Copy-on-Write semantics, snapshot creation is near-instant
- **Space Efficiency**: Built on JuiceFS clone semantics (Copy-on-Write), so snapshots avoid full data copies
- **Fast Recovery**: Restore operations quickly roll data back to a known state
- **Version Traceability**: Snapshot names and descriptions support release workflows

## Snapshot Lifecycle

```
┌─────────────────────────────────────────────────────────┐
│                      Volume                             │
│                                                         │
│    Snapshot 1          Snapshot 2          Snapshot 3   │
│    (v1.0-release)      (v1.1-beta)         (v2.0-dev)   │
│         │                   │                   │       │
│         ▼                   ▼                   ▼       │
│    ┌─────────┐         ┌─────────┐         ┌─────────┐  │
│    │ Snap A  │         │ Snap B  │         │ Snap C  │  │
│    └─────────┘         └─────────┘         └─────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
                          │
                          │ Restore
                          ▼
                   Restore to specific snapshot
```

## Create a Snapshot

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `volumeID := "vol_abc123xyz"

// Create snapshot with name only
snapshot, err := client.CreateVolumeSnapshot(ctx, volumeID, apispec.CreateSnapshotRequest{
    Name: "v1.0-release",
})

    panic(err)
}

snapshot, err = client.CreateVolumeSnapshot(ctx, volumeID, apispec.CreateSnapshotRequest{
    Name:        "v1.1-beta",

})
if err != nil {
    panic(err)
}
fmt.Printf("Snapshot created: %s (%s)\\n", snapshot.ID, snapshot.Name)
`
    },
    {
      label: "Python",
      language: "python",
      code: `import os

from sandbox0.apispec.models.create_snapshot_request import CreateSnapshotRequest

client = Client(token=os.environ["SANDBOX0_TOKEN"])

volume_id = "vol_abc123xyz"

# Create snapshot with name only

    volume_id,
    CreateSnapshotRequest(name="v1.0-release"),
)
print(f"Snapshot created: {snapshot.id}")

# Create snapshot with description
snapshot = client.volumes.create_snapshot(
    volume_id,
    CreateSnapshotRequest(

        description="Feature: Added user authentication module",
    ),
)
print(f"Snapshot created: {snapshot.id} ({snapshot.name})")`
    },
    {
      label: "TypeScript",
      language: "typescript",
      code: `import { Client } from "sandbox0";

const client = new Client({
    token: process.env.SANDBOX0_TOKEN!,
});

const volumeId = "vol_abc123xyz";

});

// Create snapshot with description

    description: "Feature: Added user authentication module",

    },

# Create snapshot with description
s0 snapshot create vol_abc123xyz \

  --description "Feature: Added user authentication module"

# For automation scripts, prefer JSON output:
s0 snapshot create vol_abc123xyz --name "v1.2-prod" -o json`

## Restore a Snapshot

Restoring a snapshot rolls the Volume data back to the point when the snapshot was created.

<Callout variant="info">
The restore API returns `200` with `{status: "restored"}` on success. Runtime failures can include lock contention (`409`) and remount timeout (`504`).
</Callout>

<Callout variant="warning">
Restore is irreversible. All data changes made after the snapshot was created are lost. Create a new backup snapshot before restoring.
</Callout>

<Tabs
  tabs={[
    {
      label: "Go",
      language: "go",
      code: `volumeID := "vol_abc123xyz"
snapshotID := "snap_def456uvw"

// Create backup snapshot before restore
_, err := client.CreateVolumeSnapshot(ctx, volumeID, apispec.CreateSnapshotRequest{
    Name: "pre-restore-backup",
})
if err != nil {
    panic(err)
}

// Execute restore
result, err := client.RestoreVolumeSnapshot(ctx, volumeID, snapshotID)
if err != nil {
    panic(err)
}
status := "restored"
if data, ok := result.Data.Get(); ok {
    status = data.Status.Or(status)
}
fmt.Printf("Volume restore status: %s\\n", status)`
    },
    {
      label: "Python",
      language: "python",
      code: `from sandbox0.apispec.models.create_snapshot_request import CreateSnapshotRequest

volume_id = "vol_abc123xyz"
snapshot_id = "snap_def456uvw"

# Create backup snapshot before restore
client.volumes.create_snapshot(
    volume_id,
    CreateSnapshotRequest(name="pre-restore-backup"),
)

# Execute restore
result = client.volumes.restore_snapshot(volume_id, snapshot_id)
print(f"Restore success: {result.success}")
if hasattr(result.data, "status"):
    print(f"Restore status: {result.data.status}")`
    },
    {
      label: "TypeScript",
      language: "typescript",
      code: `const volumeId = "vol_abc123xyz";
const snapshotId = "snap_def456uvw";

// Create backup snapshot before restore
await client.volumes.createSnapshot(volumeId, {
    name: "pre-restore-backup",
});

// Execute restore
const result = await client.volumes.restoreSnapshot(volumeId, snapshotId);
console.log("Restore success:", result.success);
if (result.data?.status) {
    console.log("Restore status:", result.data.status);
}`
    },
    {
      label: "CLI",
      language: "bash",
      code: `# Create backup snapshot before restore
s0 snapshot create vol_abc123xyz --name "pre-restore-backup"

# Execute restore
s0 snapshot restore vol_abc123xyz snap_def456uvw

# Output:
# Snapshot snap_def456uvw restored successfully`

## Delete a Snapshot

<Callout variant="info">
Delete is idempotent in the current infra implementation: deleting an already-missing snapshot still returns success (`deleted: true`).
</Callout>

    panic(err)

<Callout variant="info">
Deleting a snapshot does not affect current Volume data. The live Volume state remains intact even if all snapshots are removed.
</Callout>

## Best Practices

### Naming Conventions

<Callout variant="info">
Use predictable names so rollback points are obvious:
- Version-based: `v1.0.0`, `v1.1.0-beta`, `v2.0-rc1`
- Time-based: `2026-03-02-daily`, `before-migration`
- Event-based: `pre-deploy`, `post-cleanup`, `client-demo`
</Callout>

### Regular Cleanup

<Callout variant="warning">
Snapshots consume storage. Define retention policies such as:
- Keep the latest N snapshots
- Keep snapshots for a fixed time window
- Remove snapshots marked deprecated
</Callout>

### Backup Before Restore

<Callout variant="danger">
Restore is irreversible. Always create a new snapshot of the current state before rollback.
</Callout>
