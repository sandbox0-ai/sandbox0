# Configuration

This page focuses on the 20% of config that solves 80% of real deployments.

For full schema, use source of truth:
- `infra-operator/api/v1alpha1/sandbox0infra_types.go`
- `infra-operator/chart/crds/infra.sandbox0.ai_sandbox0infras.yaml`

## Most Important Fields

| Field | Why it matters |
|---|---|
| `spec.database` | Main metadata DB for platform services |
| `spec.storage` | JuiceFS object backend (`builtin`/`s3`/`oss`) |
| `spec.registry` | Image pull/push credential model |
| `spec.services.*.enabled` | Selects deployed topology |
| `spec.controlPlane` | Required for data-plane clusters in multi-cluster mode |
| `spec.cluster.id` | Cluster identity used by scheduler registration |
| `spec.publicExposure` | Public routing domain and region ID |
| `spec.builtinTemplates` | Seeds default templates and pool sizes |

## Minimal Single-Cluster (Recommended Start)

```yaml
apiVersion: infra.sandbox0.ai/v1alpha1
kind: Sandbox0Infra
metadata:
    name: minimal
    namespace: sandbox0-system
spec:
    version: latest
    database:
        type: builtin
        builtin:
            enabled: true
    publicExposure:
        enabled: true
        rootDomain: sandbox0.app
        regionId: aws-us-east-1
    services:
        internalGateway:
            enabled: true
            service:
                type: NodePort
                port: 30080
            config:
                authMode: public
        manager:
            enabled: true
    builtinTemplates:
        - templateId: default
          image: sandbox0ai/otemplates:default-v0.1.0
          pool:
            minIdle: 1
            maxIdle: 5
```

## Add Persistent Volume + Network Policy

```yaml
spec:
    storage:
        type: s3
        s3:
            bucket: sandbox0-prod
            region: us-east-1
            endpoint: https://s3.amazonaws.com
            credentialsSecret:
                name: aws-credentials
                accessKeyKey: accessKeyId
                secretKeyKey: secretAccessKey
    services:
        storageProxy:
            enabled: true
        netd:
            enabled: true
```

## Multi-Cluster Data Plane Essentials

```yaml
spec:
    controlPlane:
        url: https://api.sandbox0.example.com
        internalAuthPublicKeySecret:
            name: control-plane-public-key
            key: public.key
    cluster:
        id: cluster-001
        name: production-use1-a
    services:
        internalGateway:
            enabled: true
            config:
                authMode: internal
        manager:
            enabled: true
        storageProxy:
            enabled: true
```

## Service Matrix

| Service | Plane | Required |
|---|---|---|
| `internalGateway` | data plane | Yes |
| `manager` | data plane | Yes |
| `storageProxy` | data plane | Only when volume/snapshot features are needed |
| `netd` | data plane | Only when network policy features are needed |
| `edgeGateway` | control plane | Multi-cluster only |
| `scheduler` | control plane | Multi-cluster only |

## Practical Rules

- Use `builtin` database/storage only for eval or small internal environments.
- Pin `spec.version` in production instead of leaving `latest`.
- Keep control plane and data planes in the same regional storage context.
- Enable only the services required by your feature set.
