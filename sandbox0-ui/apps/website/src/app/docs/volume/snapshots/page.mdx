# Volume Snapshots

A Snapshot is a point-in-time read-only copy of a Volume. Snapshots support data backup, version management, and fast environment cloning.

## Snapshot Features

- **Instant Creation**: Based on JuiceFS Copy-on-Write technology, snapshots are created almost instantly
- **Space Efficient**: Snapshots only store differential data, not full copies
- **Fast Recovery**: Restoring Volume data from snapshots is extremely fast
- **Version Management**: Support naming and describing snapshots for easy tracking

## Snapshot Lifecycle

```
┌─────────────────────────────────────────────────────────┐
│                      Volume                             │
│                                                         │
│    Snapshot 1          Snapshot 2          Snapshot 3  │
│    (v1.0-release)      (v1.1-beta)         (v2.0-dev)   │
│         │                   │                   │       │
│         ▼                   ▼                   ▼       │
│    ┌─────────┐         ┌─────────┐         ┌─────────┐ │
│    │ Snap A  │         │ Snap B  │         │ Snap C  │ │
│    └─────────┘         └─────────┘         └─────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
                          │
                          │ Restore
                          ▼
                   Restore to specific snapshot
```

## Restore a Snapshot

Restoring a snapshot rolls the Volume data back to the state when the snapshot was created.

<Callout variant="warning">
Restore is irreversible. All data changes made after the snapshot was created will be lost. Create a new snapshot as backup before restoring.
</Callout>

<Tabs
  tabs={[
    { label: "Go", language: "go", code: "volumeID := \"vol_abc123xyz\"\nsnapshotID := \"snap_def456uvw\"\n\n// Create backup snapshot before restore\n_, err := client.CreateVolumeSnapshot(ctx, volumeID, apispec.CreateSnapshotRequest{\n    Name: \"pre-restore-backup\",\n})\nif err != nil {\n    panic(err)\n}\n\n// Execute restore\nresult, err := client.RestoreVolumeSnapshot(ctx, volumeID, snapshotID)\nif err != nil {\n    panic(err)\n}\nfmt.Printf(\"Volume restored to snapshot: %s\\n\", result.SnapshotID)" },

## Best Practices

### Naming Conventions

<Callout variant="info">
Use meaningful naming conventions for easy identification:
- Version-based: `v1.0.0`, `v1.1.0-beta`, `v2.0-rc1`
- Time-based: `2024-01-15-daily`, `before-migration`
- Event-based: `pre-deploy`, `post-cleanup`, `client-demo`
</Callout>

### Regular Cleanup

<Callout variant="warning">
Snapshots consume storage space. Regularly clean up old snapshots that are no longer needed. Consider:
- Keep the most recent N snapshots
- Keep snapshots within a specific time range
- Delete snapshots marked as deprecated
</Callout>

### Backup Before Restore

<Callout variant="danger">
Restore is irreversible. Always create a snapshot of the current state as backup before restoring.
</Callout>
